# Study Workflow

## Per-Subtopic Routine

1. Start with `learning-objectives.md`.
2. Read `core-concepts.md` and `deep-concepts.md`.
3. Study `domain-specific-examples.md`.
4. Implement or adapt the exercise from `code-level-understanding.md` and `hands-on-lab.md`.
5. Connect the topic to production through `real-world-usage.md` and `production-incident-review.md`.
6. Compare options in `architecture-decisions.md` and `system-design-connections.md`.
7. Practice interview answers from `interview-focus.md` and `interview-answer-framework.md`.
8. Trace the system in `internal-flow.md`.
9. Stress the design with `failure-scenarios.md`.
10. Build a debugging checklist from `debugging-strategies.md` and `debugging-playbook.md`.
11. Capture measurable improvements from `optimization-techniques.md`.
12. Finish with `common-misconceptions.md`, `staff-engineer-notes.md`, `validation-questions.md`, and `mastery-checklist.md`.

## Mastery Gate

Before moving to the next subtopic, you should be able to:

- Explain the concept without memorized wording.
- Draw or describe the internal flow.
- Predict edge-case behavior.
- Name production failure modes.
- Show one code example and one anti-pattern.
- Discuss trade-offs at small scale and large scale.
- Explain one domain-specific incident and one mitigation.
- Answer the staff-level ownership, migration, and operability questions.
